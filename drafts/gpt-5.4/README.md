# GPT 5.4 Drafts

This folder stores article rewrites drafted for review before any website publishing.

## Purpose

- Keep GPT 5.4 outputs separated from other model tests.
- Make it easy to compare drafts side by side.
- Preserve a review-first workflow before any CMS updates.

## Naming Convention

Use one Markdown file per article:

- `article-slug.md`

Example:

- `how-to-choose-the-best-property-manager-for-your-investment.md`

## Draft Template

Each draft should include:

1. Original article slug
2. Proposed title
3. Hub / category
4. Target audience
5. Strategic inputs used from:
   - `COMPETITIVE_INTELLIGENCE.md`
   - `SEO_COMPETITOR_PAGE_LANDSCAPE.md`
6. Rewritten article draft
7. Recommended internal links added in draft
8. Notes on legal / technical verification
9. Notes for final CMS review

## Workflow

1. Review the two research documents.
2. Review the source article.
3. Draft the rewrite here.
4. Update `SEO_PROGRESS.md`.
5. Push changes to GitHub for review.

No draft in this folder should be treated as published website content.
