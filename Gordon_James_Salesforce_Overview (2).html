Executive Summary

This document provides a comprehensive overview of Gordon James Realty's Salesforce-based property management platform. The system manages the complete lifecycle of residential and HOA/commercial properties, from advertising and tenant placement through lease management, maintenance coordination, and owner accounting.

Architectural Philosophy

The platform is modular, API-driven, and designed to operate independently of specific accounting or payment providers. Business logic lives primarily in Salesforce, while external services are replaceable via integrations. This architecture ensures flexibility and reduces vendor lock-in.

Key Capabilities

  • Complete property and lease lifecycle management
  • Automated work order processing with AI-powered summaries and invoice reading
  • Three work order workflows: client-submitted, internal direct, and project-based
  • AI-powered financial management with 80% reduction in manual data entry
  • GL Variable structure for complete client financial separation
  • Multi-level approval workflows with client threshold routing
  • Integrated tenant placement and showing coordination
  • Multi-unit building and HOA/commercial property support with contract management
  • Full accounting integration via AcctSeed and Workato
  • Automated payments processing through Forte (ACH and credit card)
  • Stripe integration for resale document fees and virtual card management
  • Expensify credit card tracking with automatic client billing
  • Weekly bank reconciliation with Bank United integration
  • Automated monthly financial close and report distribution
  • Document management, e-signatures (DocuSign & Box Sign), and automated merging
  • Customer portal access via Magentrix for clients and owners
  • AI-powered document processing with OpenAI and Google Document AI
  • Automated physical mail delivery via Lob.com

System Architecture

The Salesforce platform consists of four main layers working together:

User Interface Layer

Record pages, custom actions, and Skuid (custom pages for billing, work orders, and data entry)

Automation Layer

Automated workflows (Flows), custom business logic (Apex triggers), and AI-powered document processing

Data Layer

Custom objects for properties, leases, tenants, work orders, contracts, notices, mailings, and resale documents

Integration Layer

Connections to payments, accounting, documents, AI processing, customer portals, and physical mail delivery

Core Business Objects

Property Management Foundation

Properties & Buildings

  • Property (Property_1__c) — Individual rental units or properties
  • Building (Building__c) — Multi-unit buildings containing multiple properties
  • Links to owners (Accounts), advertising, leases, work orders, and maintenance contracts

Leases & Tenants

  • Lease (Lease_1__c) — Rental agreements connecting properties to tenants
  • Tenant (Tenant_1__c) — Current and prospective tenants
  • Tracks lease terms, renewals, notices, payment configurations, and move-in/move-out details

Move Management

  • Move (Move_1__c) — Move-in and move-out coordination
  • Record types for: Rental Move-In, Rental Move-Out, Owner Move-Out, HOA Moves
  • Connected to evaluations, key handoff, and turnover work orders

Work Orders & Maintenance

  • Work Order V2 (Work_Order_V2__c) — Primary work order object for all maintenance and jobs
  • Recurring Work Order (Recurring_Work_Order__c) — Scheduled maintenance jobs
  • Calendar Activity V2 (Calendar_Activity_V2__c) — Work order activities, close-out forms to vendors, integrated with AcctSeed for payments
  • Seasonal Maintenance (Seasonal_Maintenance__c) — Seasonal maintenance planning and tracking
  • Handles cleaning, turnover work, seasonal maintenance, repairs, and vendor billing
  • AI-powered work order summaries and invoice processing via OpenAI and Google Document AI

Tenant Placement

  • Advertising (Advertising__c) — Property listings and marketing
  • Viewings (Viewings__c) — Scheduled property showings
  • Appointment Slots — Showing time management
  • Tracks showing agents and commission payments

HOA & Property Management

  • Contract (Contract__c) — Manages contracts for HOA clients
  • Notice (Notice__c) — Generates and sends notices to clients (owners and tenants)
  • Mailing (Mailing__c) — Physical mail generation and delivery via Lob.com integration
  • Resale Document (Resale_Document__c) — Manages resale packets for property transactions
  • Accounting Workflow (Accounting_Workflow__c) — Custom object for invoice and cash receipt processing prior to AcctSeed integration; bridges AI-powered document processing with accounting approval workflows

Key Business Processes

Work Order Processing Flows

The system supports three distinct work order workflows depending on the source and complexity:

1️⃣ Client-Submitted Work Orders

Path A: Direct Assignment

For routine or emergency work

  1. Work order created (Work_Order_V2__c)
  2. AI generates summary (OpenAI)
  3. Vendor directly assigned
  4. Tasks created → Job scheduled
  5. Work completed
  6. Calendar Activity V2 close-out
  7. AI processes invoice
  8. Payment via AcctSeed/Forte
  9. Client auto-billed

Path B: Quote Required

For significant work

  1. Work order created
  2. Vendor quotes requested
  3. Quotes signed via Box Sign
  4. Sent to client via Magentrix
  5. Client approves
  6. Continue with Path A steps 3-9

2️⃣ Internal Direct Work Orders

Staff-scheduled routine maintenance

  1. User creates work order directly
  2. Vendor assigned
  3. Tasks created
  4. Standard workflow: schedule → complete → close-out → billing

3️⃣ Project-Based Work Orders

Complex projects requiring review

  1. Project record created
  2. Asset management reviews scope
  3. Project approved
  4. Quotes scheduled from vendors
  5. Best quote selected
  6. Work order created from project
  7. Vendor assigned
  8. Continue with standard workflow

Credit Card & Expense Management Process

The platform manages two types of corporate credit cards through Expensify integration:

Card Type Users Process
Chase Company Cards Senior Management Monthly reconciliation through Expensify → Synced to AcctSeed
Expensify Corporate Cards Field Maintenance Team Real-time expense tracking with receipt scanning and automatic client billing

Expensify Corporate Card Workflow:

Purchase Made:
1. Field team member uses Expensify card
2. Receipt scan REQUIRED in Expensify app
3. User tags to:
   - Property/HOA account (required)
   - Work Order (if applicable)

Billing Path A - WITH Work Order:
1. Expense tagged to work order
2. Vendor completes work + submits Calendar Activity V2 close-out
3. Accounting reviews work order
4. Material costs included in client pricing
5. Client billed for labor + materials via AcctSeed

Billing Path B - WITHOUT Work Order:
1. Expense tagged to property only
2. Weekly Expensify report submitted
3. Client AUTOMATICALLY BILLED for expense amount
4. Synced to AcctSeed via Workato

Stripe Payment & Virtual Card Process

Stripe serves two critical functions:

Function 1: Resale Document Fees

  1. Resale document request created
  2. Payment link generated via Stripe
  3. Client pays resale document fee
  4. Payment confirmation triggers processing
  5. Resale packet assembled and delivered

Function 2: Virtual Card Management

  1. User requests virtual card
  2. Selects HOA/Property + amount
  3. Approval: Auto (below threshold) or Accounting
  4. Stripe API creates SINGLE-USE card
  5. Purchase completed
  6. Card AUTOMATICALLY DEACTIVATED
  7. Receipt upload REQUIRED for next card
  8. Client automatically billed

Key Controls: One card at a time, receipt required, automatic billing, fraud prevention

Financial Management & Reporting

GL Variable Structure

Organizational Accounting Framework

  • Each HOA client and company entity has a unique GL Variable assigned in AcctSeed
  • GL Variables enable complete financial separation and reporting by client
  • All transactions, invoices, and payments tagged to specific GL Variables
  • Financial reports generated independently for each GL Variable

Accounting Landing Page

Custom Skuid-based interface providing accounting team and account managers with centralized access to:

  • Pending invoice/cash receipt approvals
  • Payables ready for review
  • Payables ready to be paid
  • Bank reconciliation status
  • Financial report generation tools
  • Approval thresholds and client approval requirements

Invoice & Cash Receipt Processing Workflow

End-to-End Automated Processing:

1. Document Receipt & Scanning:
   - Invoices/checks received via mail
   - Office staff scans documents
   - Uploaded to designated Box folder

2. AI-Powered Document Processing (Workato orchestration):
   - Box upload triggers Workato workflow
   - Google Document AI extracts structured data
     (vendor, invoice #, date, amounts, terms)
   - OpenAI interprets non-structured data
   - Pinecone + Cohere match vendor/client to Salesforce
   - High-confidence matching reduces manual entry

3. Accounting Workflow Record Creation:
   - Accounting_Workflow__c custom object created
   - Contains parsed data, matched vendor/client, GL Variable
   - Serves as review queue before AcctSeed entry

4. Accounting Team Review & Approval:
   - Team reviews on landing page
   - Validates AI matching and data
   - Approves → Creates AcctSeed Cash Receipt or Payable

5. Account Manager Review (Payables only):
   - Views "payables ready to be reviewed"
   - Reviews business purpose and GL Variable
   - Client Approval Threshold Check:
     → Below threshold: Account manager approves
     → Above threshold: Routed to client via Magentrix
   - After approval → "payables ready to be paid"

6. Payment Processing:
   - Accounting manager final approval
   - Payment triggered:
     → eCheck via Forte
     → Physical check manually printed

Key Benefits

  • AI reduces manual data entry by ~80%
  • Vector embeddings ensure accurate vendor matching
  • Multi-level approval workflow prevents unauthorized payments
  • Client approval thresholds embedded in process
  • Complete audit trail from scanned document to payment

Weekly Bank Reconciliation Process

Bank United Integration:

Step Process
1. Transaction Download Mass download from Bank United → CSV upload to Salesforce
2. Transaction Matching Team matches existing AcctSeed transactions or creates new ones
3. Exception Handling Unmatched transactions flagged → Investigation → Manual matching or journal entries

Monthly Financial Close & Reporting

10th of Month: Report Generation

  1. Financial Report Process Execution
    • Accounting team runs process in Salesforce
    • AcctSeed generates PDF statements by GL Variable:
      • Income statement (P&L)
      • Balance sheet
      • Cash flow statement
      • General ledger detail
      • AR/AP aging
    • Reports stored in Box, linked to accounts
  2. Report Review
    • Accounting team validates accuracy
    • Confirms GL Variable assignments
    • Verifies all transactions posted

15th of Month: Automatic Distribution

  1. Automated Client Delivery
    • Reports automatically sent to clients
    • Email with Box shared links
    • Access via Magentrix portal
    • Notifications to account managers

Pre-Paid Invoice Processing

Automated Pre-Payment Tracking:

Process Flow:
1. Pre-paid invoices uploaded to Box folder → Triggers Workato

2. AI Document Processing:
   - Google Document AI reads invoice details
   - OpenAI interprets payment terms
   - Pinecone + Cohere matches to Salesforce records
   - Updates Salesforce with pre-payment info

3. Transaction Matching:
   - System searches for matching AcctSeed transactions
   - Links pre-paid invoice to transaction
   - Updates GL coding

4. Month-End Reconciliation:
   - Unmatched transactions flagged
   - Accounting team manually reviews
   - Ensures all pre-payments applied
   - Prevents duplicate payment

Financial Controls Summary

  • GL Variable Separation: Complete financial isolation by client
  • AI-Powered Processing: 80% reduction in manual data entry
  • Multi-Level Approvals: Accounting team → Account manager → Client (threshold-based)
  • Weekly Bank Rec: Continuous reconciliation prevents month-end surprises
  • Monthly Close: Automated report generation and distribution
  • Pre-Payment Tracking: Prevents duplicate billing and ensures proper application
  • Complete Audit Trail: From scanned document to payment to financial statement
